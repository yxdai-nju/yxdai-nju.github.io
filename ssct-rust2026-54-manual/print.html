配置 SSH

在你本地电脑的 ~/.ssh/config 中添加以下内容，把 XX 替换成你的编号（如 01、09、10）：

Host rust2026
  HostName 114.212.189.54
  Port 322XX
  User rust
  StrictHostKeyChecking no
  UserKnownHostsFile /dev/null

Host rebuild
  HostName 114.212.189.54
  Port 22
  User stuXX

配置完成后，你可以使用两个命令：

  • ssh rust2026：进入你的工作区容器，日常开发在这里进行
  • ssh rebuild：触发容器重建（修改 Dockerfile 后执行）

安全提醒

部分同学从原 Rust2024 机器迁移过来，~/.ssh/authorized_keys 文件中可能残留其他同学的公钥。请尽快检查该文件，只保留你自己的公钥。

环境说明

你通过 ssh rust2026 进入的是一个运行在服务器上的容器，称为“工作区容器”。这个容器由你 home 目录下的 Dockerfile 定义，你可以自由定制其中的软件环境。

容器内预装了常用软件，包括 git、vim、tmux、htop 等。同时预装了 build-essential，包含 gcc、g++、make 等编译工具链，可用于编译 C/C++ 项目和 Rust 项目的本地依赖。容器内还预装了 kubectl 和 docker 命令行工具，方便你部署服务和构建镜像，后续章节会详细介绍它们的用法。

哪些内容会保留

以下目录和文件在容器重建后保留，不会丢失：

路径用途
~/workspace/存放项目和代码
~/workspace/.container/存放 Dockerfile 和构建所需文件
~/.ssh/SSH 密钥和配置
~/.gnupg/GPG 密钥
~/.kube/Kubernetes 配置
~/.vscode-server/VS Code Remote 插件和设置
~/.bash_history命令历史
/data/存放数据

这些目录的数据存储在主机上，容器重建不会影响它们。服务器每晚凌晨 3 点自动创建快照备份，保留最近 30 天的快照。如果误删文件，可以微信联系戴宇轩请求恢复。

哪些内容不会保留

容器重建后，以下内容会丢失：

  • 运行中的进程（包括 tmux、screen 会话）
  • 上述目录之外的文件
  • 通过 apt install 直接安装但未写入 Dockerfile 的软件

如果你需要某个软件在重建后仍然存在，请将安装命令写入 Dockerfile。

网络地址

地址含义典型用途
127.0.0.1工作区容器内部工作区内进程间通信
10.42.0.1服务器主机访问你用 Docker 启动的服务
Service 名称（如 my-postgres）*你部署的服务访问用 kubectl 部署的服务

* kubectl 是 Kubernetes 的命令行工具，后续章节会详细介绍如何用它部署服务。

定制环境

直接在容器里执行 apt install 或 pip install 可以临时安装软件，但容器重建后这些软件会消失。把安装命令写入 Dockerfile，每次重建都会自动执行，环境始终保持一致。

这样做还有额外好处：你可以随意折腾，随便安装东西、删系统文件，不用担心后果——重建一下就恢复了。当你想配置较复杂的环境时，可以把 Dockerfile 发给大模型，让它帮忙修改。重建后即可得到新环境，出问题可退回到之前的版本重来。

Dockerfile 位置

Dockerfile 位于 ~/workspace/.container/Dockerfile。你可以在它的目录下放置其他文件，然后在 Dockerfile 中用 COPY 指令复制到容器内。

例如你下载了一个 .deb 安装包（qtspim_9.1.24_linux64.deb）想要安装，可以把它放到 ~/workspace/.container/ 目录下，然后在 Dockerfile 中添加：

COPY qtspim_9.1.24_linux64.deb /tmp/qtspim_9.1.24_linux64.deb
RUN apt-get update \
    && apt-get install -y /tmp/qtspim_9.1.24_linux64.deb \
    && apt-get clean \
    && rm -rf /var/lib/apt/lists/* \
    && rm -f /tmp/qtspim_9.1.24_linux64.deb

重建流程

修改 Dockerfile 后，在你的本地电脑上（平时执行 ssh rust2026 所用的电脑）执行：

ssh rebuild

这会触发镜像构建、推送和容器更新。构建完成后你的 SSH 连接会断开，稍等片刻重新连接即可。

完整示例

以下是文档编写者自己在用的 Dockerfile 模板。你可以在此基础上修改，删除不需要的部分或添加自己的工具。

# 基础镜像，可以换成 Ubuntu、Alpine 等
FROM docker.io/library/debian:trixie-20251229

ARG USERNAME="rust"
ARG USER_UID
ARG USER_GID

# ============================================================================
# 系统软件包/常用软件包
# ============================================================================

ARG DEBIAN_FRONTEND="noninteractive"
RUN apt-get update \
    && apt-get install -y \
        ca-certificates curl gnupg locales openssh-server sudo vim \
        build-essential libssh-dev pkg-config \
        git redis-tools \
        htop numactl jq rsync tmux tree \
    && apt-get clean \
    && rm -rf /var/lib/apt/lists/*

# 添加 Docker 仓库
RUN install -m 0755 -d /etc/apt/keyrings \
    && curl -fsSL https://download.docker.com/linux/debian/gpg -o /etc/apt/keyrings/docker.asc \
    && chmod a+r /etc/apt/keyrings/docker.asc \
    && echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] \
        https://download.docker.com/linux/debian trixie stable" \
        | tee /etc/apt/sources.list.d/docker.list

# 添加 Kubernetes 仓库（v1.33，与服务器版本一致）
RUN curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.33/deb/Release.key \
        | gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg \
    && echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] \
        https://pkgs.k8s.io/core:/stable:/v1.33/deb/ /' \
        | tee /etc/apt/sources.list.d/kubernetes.list

# 添加 Helm 仓库
RUN curl -fsSL https://packages.buildkite.com/helm-linux/helm-debian/gpgkey \
        | gpg --dearmor | tee /usr/share/keyrings/helm.gpg > /dev/null \
    && echo "deb [signed-by=/usr/share/keyrings/helm.gpg] \
        https://packages.buildkite.com/helm-linux/helm-debian/any/ any main" \
        | tee /etc/apt/sources.list.d/helm-stable-debian.list

# 添加 PostgreSQL 仓库
RUN install -d /usr/share/postgresql-common/pgdg \
    && curl -fsSL https://www.postgresql.org/media/keys/ACCC4CF8.asc \
        | gpg --dearmor -o /usr/share/postgresql-common/pgdg/apt.postgresql.org.gpg \
    && echo "deb [signed-by=/usr/share/postgresql-common/pgdg/apt.postgresql.org.gpg] \
        https://apt.postgresql.org/pub/repos/apt trixie-pgdg main" \
        | tee /etc/apt/sources.list.d/pgdg.list

# 安装容器和数据库工具
RUN apt-get update \
    && apt-get install -y \
        docker-ce-cli docker-compose-plugin helm kubectl \
        postgresql-client-18 \
    && apt-get clean \
    && rm -rf /var/lib/apt/lists/*

# ============================================================================
# 用户和系统配置（一般不需要修改）
# ============================================================================

RUN groupadd -g ${USER_GID} ${USERNAME} \
    && useradd -m -s /bin/bash -u ${USER_UID} -g ${USER_GID} ${USERNAME} \
    && echo ${USERNAME} ALL=\(root\) NOPASSWD:ALL > /etc/sudoers.d/${USERNAME} \
    && chmod 0440 /etc/sudoers.d/${USERNAME}
ENV HOME="/home/${USERNAME}"

RUN mkdir -p ${HOME}/.config ${HOME}/.local \
    && chown -R ${USERNAME}:${USERNAME} ${HOME}/.config ${HOME}/.local

RUN echo 'export GPG_TTY=$(tty)' >> ${HOME}/.bashrc

USER ${USERNAME}
RUN mkdir -p /home/${USERNAME}/.docker \
    && echo "export DOCKER_HOST=unix:///home/${USERNAME}/.docker/docker.sock" >> /home/${USERNAME}/.bashrc
USER root

RUN sed -i '/en_US.UTF-8/s/^# //g' /etc/locale.gen \
    && locale-gen \
    && ln -sf /usr/share/zoneinfo/Asia/Shanghai /etc/localtime \
    && echo "Asia/Shanghai" > /etc/timezone \
    && dpkg-reconfigure -f noninteractive tzdata \
    && echo "TZ=Asia/Shanghai" >> /etc/environment \
    && echo "LANG=en_US.UTF-8" >> /etc/environment \
    && echo "LC_ALL=en_US.UTF-8" >> /etc/environment

# ============================================================================
# 个人开发环境（主要在这里定制）
# ============================================================================

# 安装自己需要的额外软件包
# RUN apt-get update \
#     && apt-get install -y \
#         packages-that-you-need \
#     && apt-get clean \
#     && rm -rf /var/lib/apt/lists/*

USER ${USERNAME}
WORKDIR ${HOME}

# 通过 Nix 安装自己需要的额外软件包（可选，用于安装一些 apt 未提供的软件包）
# 这里安装 buck2、reindeer 作为示例

# 安装 Rust 工具链
# 可通过 cargo install 安装额外软件包，这里安装 mdbook、mdbook-mermaid 作为示例

    && rm -rf ${HOME}/.cargo/registry ${HOME}/.cargo/git

    && conda config --add channels conda-forge \
    && conda create -n common python=3.13

常见定制场景

安装 Rust nightly 工具链并设为默认：

RUN rustup toolchain install nightly \
    && rustup default nightly \
    && rustup component add rust-src

使用 lld 作为链接器：

可加快 Rust 项目的链接速度。首先确保安装了 lld。在“系统软件包/常用软件包”部分的 apt-get install 中添加 lld。然后在“个人开发环境”部分添加：

RUN echo 'export RUSTFLAGS="-Clink-args=-fuse-ld=lld"' >> ${HOME}/.bashrc

RUN conda run -n common pip install numpy pandas torch

示例：部署数据库

如果你的项目需要一个数据库（PostgreSQL、Redis、MongoDB 等），并且希望它一直运行、崩溃后自动重启、数据在重启后保留，那么用 kubectl 部署是合适的选择。对于临时任务（跑实验、调试脚本），用 tmux 更简单。

本章以部署 PostgreSQL 为例，介绍如何在你的命名空间内部署一个长期运行的服务。

前置准备

上一章的 Dockerfile 示例中，“系统软件包/常用软件包“部分安装了 postgresql-client-18，它提供 psql 命令行工具。本章假定你的容器中已安装该软件包。

编写部署文件

创建文件 ~/workspace/postgres.yaml：

apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: postgres-data
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 50Gi
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: postgres
spec:
  replicas: 1
  selector:
    matchLabels:
      app: postgres
  template:
    metadata:
      labels:
        app: postgres
    spec:
      containers:
      - name: postgres
        image: postgres:18
        env:
        - name: POSTGRES_PASSWORD
          value: "passwd"
        - name: POSTGRES_USER
          value: "rust"
        - name: POSTGRES_DB
          value: "app"
        ports:
        - containerPort: 5432
        volumeMounts:
        - name: data
          mountPath: /var/lib/postgresql
      volumes:
      - name: data
        persistentVolumeClaim:
          claimName: postgres-data
---
apiVersion: v1
kind: Service
metadata:
  name: my-postgres
spec:
  selector:
    app: postgres
  ports:
  - port: 5432
    targetPort: 5432

这个文件定义了三个资源：

  • PersistentVolumeClaim：申请 50GB 持久化存储，数据库文件存于此处，容器重启不会丢失
  • Deployment：声明运行一个 PostgreSQL 容器，挂载上述存储
  • Service：为数据库提供固定的访问地址 my-postgres

部署

执行部署：

kubectl apply -f ~/workspace/postgres.yaml

查看状态：

kubectl get pods,svc,pvc

这条命令同时查询三种资源：Pod（运行中的容器）、Service（简写 svc，服务访问入口）、PersistentVolumeClaim（简写 pvc，持久化存储）。你也可以分别查询，如 kubectl get pods 或 kubectl get svc。

等待 Pod 状态变为 Running，PVC 状态变为 Bound。

连接数据库

Service 名称 my-postgres 既是名字，也是访问这个服务的地址。如果你写程序来连接这个数据库，可以直接使用连接字符串 postgres://rust:passwd@my-postgres:5432/app。

这里用 psql 命令行工具来连接：

PGPASSWORD=passwd psql -h my-postgres -U rust -d app

连接成功后，执行 \dt+ 查看表列表（新数据库为空），按 Ctrl+D 退出。

常用命令

命令作用
kubectl apply -f xxx.yaml创建或更新资源
kubectl get pods查看 Pod 列表
kubectl get svc查看 Service 列表
kubectl get pvc查看持久化存储
kubectl logs <pod名>查看日志
kubectl logs -f <pod名>实时查看日志
kubectl describe pod <pod名>查看详细信息和事件
kubectl exec -it <pod名> -- <命令>在容器内执行命令
kubectl delete -f xxx.yaml删除资源

删除数据库

kubectl delete svc my-postgres
kubectl delete deployment postgres
# kubectl delete pvc postgres-data

若只删除 Deployment 和 Service 但保留 PVC，数据不会丢失，下次重新 kubectl apply 时数据仍在。

如果想连数据一起删除，还需执行上面被注释掉的 kubectl delete pvc postgres-data。也可以用 kubectl delete -f ~/workspace/postgres.yaml 效果与执行上述三条命令相同。删除 PVC 前请确认数据已备份或确实不再需要。

常见问题

如果连接数据库时出现问题，首先执行 kubectl get pods 查看 Pod 状态。

现象可能原因解决方法
Pod 状态 PendingPVC 未绑定或资源不足kubectl describe pod <pod名> 查看事件
Pod 状态 CrashLoopBackOff容器启动后崩溃kubectl logs <pod名> 查看错误信息

示例：部署Web应用

本章以一个最简单的 Flask 应用为例，演示如何构建镜像、推送到仓库、部署到集群并从外网访问。

整体流程

flowchart LR
    Code["编写代码"] --> Build["构建镜像"]
    Build --> Push["推送到仓库"]
    Push --> Deploy["kubectl apply"]
    Deploy --> Test["测试访问"]
    Test -->|"需要修改"| Code

项目结构

~/workspace/myapp/
├── log/
│   └── visitors.log
├── app.py
├── Dockerfile
├── deployment.yaml
└── deploy.sh

编写应用

创建项目目录和日志目录：

mkdir -p ~/workspace/myapp/log
cd ~/workspace/myapp

创建 app.py：

import os
from flask import Flask, request

app = Flask(__name__)

@app.route('/greeting')
def greeting():
    name = request.args.get('name', 'World')
    ip = request.remote_addr

    log_dir = '/log'
    log_file = os.path.join(log_dir, 'visitors.log')

    with open(log_file, 'a') as f:
        f.write(f'{name} from {ip}\n')

    return f'Hello, {name}!\n'

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=8080)

创建 Dockerfile。注意把 11XX 替换成你的 UID/GID（如 stu02 对应 1102）：

FROM python:3.13-slim
WORKDIR /app
RUN pip install --no-cache-dir flask
COPY app.py .
RUN useradd -u 11XX appuser
USER appuser
EXPOSE 8080
CMD ["python", "app.py"]

编写部署文件

创建 deployment.yaml。文件中有四处需要替换：把 stuXX 替换成你的账号（如 stu02），把 11XX 替换成对应的 UID/GID（如 1102）。

apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
spec:
  replicas: 1
  selector:
    matchLabels:
      app: myapp
  template:
    metadata:
      labels:
        app: myapp
    spec:
      securityContext:
        runAsUser: 11XX      # 替换 1
        runAsGroup: 11XX     # 替换 2
        fsGroup: 11XX        # 替换 3
      containers:
        - name: myapp
          image: registry.io:5000/myapp:latest
          imagePullPolicy: Always
          ports:
            - containerPort: 8080
          volumeMounts:
            - name: log
              mountPath: /log
      volumes:
        - name: log
          hostPath:
            path: /home/stuXX/workspace/myapp/log  # 替换 4
            type: Directory
---
apiVersion: v1
kind: Service
metadata:
  name: myapp-svc
spec:
  type: NodePort
  selector:
    app: myapp
  ports:
    - port: 8080
      targetPort: 8080
      nodePort: 30080

说明：

  • registry.io:5000：宿主机上已配置的本地镜像仓库，用于存放你构建的镜像
  • imagePullPolicy: Always：每次部署都拉取最新镜像
  • type: NodePort：在服务器的指定端口暴露服务，可从外网访问
  • nodePort: 30080：服务器上的端口号，范围 30000–32767，如果与其他同学冲突就换一个
  • runAsUser、runAsGroup、fsGroup：指定容器运行时的用户和组，确保挂载目录中创建的文件拥有正确的权限，你才能在工作区容器中读写这些文件

关于目录挂载：你在工作区容器中看到的 /home/rust/workspace 目录实际上是从宿主机的 /home/stuXX/workspace 映射来的。在 deployment.yaml 中挂载目录时，需要使用宿主机上的真实路径。可用于挂载的目录有两组：

工作区容器中的路径宿主机上的真实路径
~/workspace/<子路径>/home/stuXX/workspace/<子路径>
/data/<子路径>/data/stuXX/data/<子路径>

编写部署脚本

创建 deploy.sh：

#!/bin/bash
set -ex

cd ~/workspace/myapp

docker build -t registry.io:5000/myapp:latest .
docker push registry.io:5000/myapp:latest

kubectl apply -f deployment.yaml
kubectl rollout restart deployment/myapp
kubectl rollout status deployment/myapp

赋予执行权限：

chmod +x deploy.sh

部署

./deploy.sh

脚本会依次构建镜像、推送到本地仓库、应用部署配置、重启并等待就绪。

可选：使用 ClusterIP 替代 NodePort

如果不想在公网暴露端口，可以把 Service 的 type: NodePort 改为 ClusterIP，并删除 nodePort 行：

apiVersion: v1
kind: Service
metadata:
  name: myapp-svc
spec:
  type: ClusterIP
  selector:
    app: myapp
  ports:
    - port: 8080
      targetPort: 8080

然后在本地电脑上执行 SSH 端口转发：

ssh -N rust2026 -L 8080:myapp-svc:8080

保持这个终端运行，然后访问 http://localhost:8080/greeting?name=Rustacean。这种方式更安全，适合开发调试阶段。
