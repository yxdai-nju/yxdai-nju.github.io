SSCT Rust2026 #54 服务器手册

示例：部署Web应用

本章以一个最简单的 Flask 应用为例，演示如何构建镜像、推送到仓库、部署到集群并从外网访问。

整体流程

flowchart LR
    Code["编写代码"] --> Build["构建镜像"]
    Build --> Push["推送到仓库"]
    Push --> Deploy["kubectl apply"]
    Deploy --> Test["测试访问"]
    Test -->|"需要修改"| Code

项目结构

~/workspace/myapp/
├── log/
│   └── visitors.log
├── app.py
├── Dockerfile
├── deployment.yaml
└── deploy.sh

编写应用

创建项目目录和日志目录：

mkdir -p ~/workspace/myapp/log
cd ~/workspace/myapp

创建 app.py：

import os
from flask import Flask, request

app = Flask(__name__)

@app.route('/greeting')
def greeting():
    name = request.args.get('name', 'World')
    ip = request.remote_addr

    log_dir = '/log'
    log_file = os.path.join(log_dir, 'visitors.log')

    with open(log_file, 'a') as f:
        f.write(f'{name} from {ip}\n')

    return f'Hello, {name}!\n'

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=8080)

创建 Dockerfile。注意把 11XX 替换成你的 UID/GID（如 stu02 对应 1102）：

FROM python:3.13-slim
WORKDIR /app
RUN pip install --no-cache-dir flask
COPY app.py .
RUN useradd -u 11XX appuser
USER appuser
EXPOSE 8080
CMD ["python", "app.py"]

编写部署文件

创建 deployment.yaml。文件中有四处需要替换：把 stuXX 替换成你的账号（如 stu02），把 11XX 替换成对应的 UID/GID（如 1102）。

apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
spec:
  replicas: 1
  selector:
    matchLabels:
      app: myapp
  template:
    metadata:
      labels:
        app: myapp
    spec:
      securityContext:
        runAsUser: 11XX      # 替换 1
        runAsGroup: 11XX     # 替换 2
        fsGroup: 11XX        # 替换 3
      containers:
        - name: myapp
          image: registry.io:5000/myapp:latest
          imagePullPolicy: Always
          ports:
            - containerPort: 8080
          volumeMounts:
            - name: log
              mountPath: /log
      volumes:
        - name: log
          hostPath:
            path: /home/stuXX/workspace/myapp/log  # 替换 4
            type: Directory
---
apiVersion: v1
kind: Service
metadata:
  name: myapp-svc
spec:
  type: NodePort
  selector:
    app: myapp
  ports:
    - port: 8080
      targetPort: 8080
      nodePort: 30080

说明：

  • registry.io:5000：宿主机上已配置的本地镜像仓库，用于存放你构建的镜像
  • imagePullPolicy: Always：每次部署都拉取最新镜像
  • type: NodePort：在服务器的指定端口暴露服务，可从外网访问
  • nodePort: 30080：服务器上的端口号，范围 30000–32767，如果与其他同学冲突就换一个
  • runAsUser、runAsGroup、fsGroup：指定容器运行时的用户和组，确保挂载目录中创建的文件拥有正确的权限，你才能在工作区容器中读写这些文件

关于目录挂载：你在工作区容器中看到的 /home/rust/workspace 目录实际上是从宿主机的 /home/stuXX/workspace 映射来的。在 deployment.yaml 中挂载目录时，需要使用宿主机上的真实路径。可用于挂载的目录有两组：

工作区容器中的路径宿主机上的真实路径
~/workspace/<子路径>/home/stuXX/workspace/<子路径>
/data/<子路径>/data/stuXX/data/<子路径>

编写部署脚本

创建 deploy.sh：

#!/bin/bash
set -ex

cd ~/workspace/myapp

docker build -t registry.io:5000/myapp:latest .
docker push registry.io:5000/myapp:latest

kubectl apply -f deployment.yaml
kubectl rollout restart deployment/myapp
kubectl rollout status deployment/myapp

赋予执行权限：

chmod +x deploy.sh

部署

./deploy.sh

脚本会依次构建镜像、推送到本地仓库、应用部署配置、重启并等待就绪。

测试

如果你在校园网内，可以直接用浏览器访问：

http://114.212.189.54:30080/greeting?name=Rustacean

预期看到：

Hello, Rustacean!

访问后查看 ~/workspace/myapp/log/visitors.log，应该能看到刚才的访问记录。每次访问都会追加一条日志，记录访问者的 name 参数和 IP 地址。

更新代码

修改 app.py 后，再次执行 ./deploy.sh 即可。

排查问题

kubectl get pods                      # 查看状态
kubectl describe pod <pod名>          # 查看详细信息和事件
kubectl logs <pod名>                  # 查看日志
kubectl logs -f <pod名>               # 实时日志
kubectl exec -it <pod名> -- /bin/bash # 进入容器调试
现象可能原因解决方法
ImagePullBackOff镜像名错误或未推送检查镜像名，重新 docker push
CrashLoopBackOff应用启动崩溃kubectl logs 查看错误
无法访问 NodePort端口被占用换一个 nodePort 数值

可选：使用 ClusterIP 替代 NodePort

如果不想在公网暴露端口，可以把 Service 的 type: NodePort 改为 ClusterIP，并删除 nodePort 行：

apiVersion: v1
kind: Service
metadata:
  name: myapp-svc
spec:
  type: ClusterIP
  selector:
    app: myapp
  ports:
    - port: 8080
      targetPort: 8080

然后在本地电脑上执行 SSH 端口转发：

ssh -N rust2026 -L 8080:myapp-svc:8080

保持这个终端运行，然后访问 http://localhost:8080/greeting?name=Rustacean。这种方式更安全，适合开发调试阶段。
